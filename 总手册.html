华为 HCIE-Datacom 学习手册

适用对象：零基础或基础薄弱，希望系统学习网络知识，并朝华为 HCIE-Datacom 方向进阶的学习者 目标：建立从网络基础、工程实践、故障排查到 Datacom 高阶方案设计与实验能力的完整体系 版本说明：本手册的认证路径与知识点分布参考华为官方公开考试信息与大纲；训练方法、案例拆解、学习节奏与技巧为经验性建议。

1. 先说明白：你要考的是什么

1.1 华为认证路径

华为职业认证分为三层：

  • HCIA：入门
  • HCIP：进阶
  • HCIE：专家

如果你想成为高级网络专家，建议按下面路径走：

  1. 先掌握 HCIA-Datacom 的基础体系
  2. 再补齐 HCIP 级的路由交换、园区、广域、自动化
  3. 最后冲 HCIE-Datacom 的笔试和实验

1.2 HCIE-Datacom 核心方向

根据华为官方 HCIE-Datacom V1.0 考试大纲，笔试/实验覆盖五大块：

模块笔试占比实验占比你应该怎么理解
高级路由交换技术37%40%核心中的核心，必须最强
园区网络规划与部署23%20%企业园区、CloudCampus、NAC、业务随行、VXLAN/EVPN
广域互联网络规划与部署8%5%SD-WAN、WAN 互联与可靠性
广域承载网络规划与部署15%25%MPLS、SR、SRv6、CloudWAN、SLA
网络自动化17%10%Python、SSH、NETCONF/YANG、RESTCONF、Telemetry

结论：

  • 真正拉开差距的是：高级路由交换 + 园区虚拟化 + MPLS/SR + 自动化
  • 通过 HCIE 不只是会敲命令，而是会设计、部署、验证、排障、割接

2. 零基础如何起步：先建立正确的“网络脑图”

很多人学不进去，不是因为难，而是顺序错了。

2.1 正确顺序

不要上来就学 BGP、MPLS、EVPN。 先按下面顺序建立网络脑图：

  1. 网络是什么：设备、接口、链路、报文、协议
  2. 主机如何通信：ARP、IP、路由、TCP/UDP
  3. 二层怎么通：交换、VLAN、STP、链路聚合
  4. 三层怎么通：静态路由、OSPF、BGP
  5. 怎么保证稳定：冗余、收敛、故障切换
  6. 怎么保证安全：ACL、AAA、NAT、基础安全
  7. 怎么规模化：MPLS、VPN、EVPN、VXLAN、SR
  8. 怎么管理：日志、NTP、SNMP、Telemetry、API
  9. 怎么落地：设计、割接、验证、排障、自动化

2.2 你必须先形成三个核心问题

学每个知识点时，都问自己：

  1. 它解决什么问题？
  2. 它为什么比旧方案更好？
  3. 它出故障时，我从哪里开始排查？

这是从“会配置”走向“会工程”的分水岭。

5. 从零开始的基础知识图谱

5.1 数据通信与网络基础

必须掌握：

  • OSI 七层、TCP/IP 四层
  • 封装与解封装
  • MAC、IP、端口
  • 广播域、冲突域
  • ARP 工作过程
  • TCP 三次握手/四次挥手
  • UDP 与 TCP 对比
  • 路由器和交换机的工作差异

实操案例 1：抓包理解 ARP 和 ICMP

目标：

  • 看见“主机第一次 ping 网关时到底发了什么”

步骤：

  1. 两台 PC + 一个二层交换机
  2. 配同网段地址
  3. 清空 ARP 表
  4. 开 Wireshark 抓包
  5. 执行 ping

你应该观察到：

  • 先发 ARP Request 广播
  • 对端返回 ARP Reply
  • 随后才有 ICMP Echo

思路技巧：

  • 学协议时，不要只背定义，必须抓包
  • 抓包是把抽象协议变成直观现实的最短路径

5.2 IPv4 与子网划分

必须掌握：

  • IP 地址分类与 CIDR
  • 子网掩码
  • 网络地址、广播地址、可用主机范围
  • VLSM
  • 路由汇总

实操案例 2：部门网段规划

场景：

  • 总公司 3 个部门：研发 120 人、财务 30 人、市场 50 人
  • 你有一个网段 192.168.10.0/24

要求：

  • 合理划分子网
  • 保证扩展性

标准思路：

  • 研发：/25
  • 市场：/26
  • 财务：/27
  • 预留：剩余地址段

独特技巧：

  • 子网规划先按最大网段切
  • 不要只满足“现在够用”，至少留 20%-30% 余量
  • 学会把地址规划和组织架构、楼层、业务域关联

6. 二层交换：你要学到“为什么会环路”

6.1 必学主题

  • VLAN
  • Trunk
  • STP/RSTP/MSTP
  • Eth-Trunk
  • 堆叠
  • 交换机基础安全

图示：VLAN 隔离

PC1(VLAN10) -- SW1 -- SW2 -- PC2(VLAN10)
PC3(VLAN20) -- SW1 -- SW2 -- PC4(VLAN20)

核心理解：

  • VLAN 是逻辑隔离广播域
  • Trunk 用来在设备间承载多个 VLAN

实操案例 3：VLAN + Trunk + 三层网关

目标：

  • 让 VLAN10 和 VLAN20 既隔离又可互访

典型步骤（华为 VRP 思路）：

  1. 创建 VLAN 10/20
  2. 把接入口加入不同 VLAN
  3. 上联口设为 trunk
  4. 在三层交换机上创建 Vlanif10/Vlanif20
  5. 给网关接口配置地址

验证：

  • 同 VLAN 能互通
  • 跨 VLAN 通过三层网关互通

排障思路：

  • 先看接口 UP/DOWN
  • 再看 VLAN 是否创建
  • 再看端口是否入对 VLAN
  • 再看 trunk 是否放行 VLAN
  • 最后看网关 IP 是否正确

实操案例 4：STP 环路与收敛

场景：

  • 三台交换机组成三角形

目标：

  • 理解为什么不用 STP 会广播风暴
  • 看根桥选举、根端口、指定端口、阻塞端口

独特技巧：

  • 学 STP 不要只记状态，重点是“谁离根更近”
  • 实验时故意拔掉根桥链路，观察拓扑变化

7. 三层路由：从“能通”走向“稳定地通”

7.1 必学主题

  • 静态路由
  • 缺省路由
  • 浮动静态路由
  • OSPF
  • 路由引入
  • 路由汇总
  • IPv6 基础路由

实操案例 5：静态路由 + 备份链路

目标：

  • 主链路故障时自动切到备链路

方法：

  • 主路由缺省静态
  • 备路由静态设置更大优先级

你要理解：

  • 这不是“高端技术”，但是真实工程里很常用
  • 很多网络事故，不是因为协议不会，而是优先级没设计好

实操案例 6：OSPF 单区域到多区域

步骤建议：

  1. 先做单区域
  2. 再做 Area 0 + Area 1/2
  3. 再加认证、汇总、缺省路由发布、路由过滤
  4. 再做快速收敛与排障

关键命令观察点：

  • 邻居状态
  • LSDB
  • 路由表
  • DR/BDR

排障口诀：

  • 邻居不起：先看三层可达
  • 三层可达再看 Area、Hello/Dead、认证、MTU
  • 邻居起来了但没路由：看 LSA、路由策略、汇总、引入

8. BGP：网络工程师的真正分水岭

8.1 先理解四件事

  • 为什么 IGP 不适合承载大规模自治域间路由
  • BGP 的邻居建立和路由传播逻辑
  • 路径属性如何影响选路
  • 路由控制如何落地到企业网络

8.2 必学主题

  • EBGP / IBGP
  • Next-Hop
  • Local Preference
  • MED
  • AS-Path
  • Route-Reflector
  • Community
  • 路由策略
  • BGP 与 MPLS VPN

实操案例 7：双出口企业 BGP 选路

场景：

  • 企业有两条 Internet 出口
  • 希望主用出口 A，备份出口 B
  • 某些业务走 B 更便宜

学习目标：

  • 用 Local Preference 控制出向
  • 用 AS-Path prepend / MED / community 配合控制入向

独特技巧：

  • BGP 不要背“哪个属性优先级更高”完事
  • 要把它翻译成业务语言：
  • 哪条路更便宜
  • 哪条路更稳定
  • 哪条路适合走哪类流量

9. MPLS / VPN / SR：HCIE 的硬门槛

9.1 为什么难

因为它同时要求你理解：

  • 标签转发
  • 控制层与转发层分离
  • VPN 路由隔离
  • 大规模承载网设计

9.2 必学主题

  • MPLS 基础
  • LDP
  • L3VPN
  • 跨域 Option A/B/C
  • EVPN
  • Segment Routing
  • SRv6

图示：MPLS L3VPN 基本模型

CE1 -- PE1 == P == PE2 -- CE2

理解顺序：

  1. 先理解普通 IP 转发
  2. 再理解“为什么要打标签”
  3. 再理解“为什么 VPN 要有 RD/RT”
  4. 最后才看跨域和复杂选项

实操案例 8：MPLS L3VPN 基础

目标：

  • CE1 与 CE2 在不同站点互通
  • 承载网核心只看标签，不看用户私网细节

你必须会：

  • 搭 IGP
  • 配 MPLS/LDP
  • 建 MP-BGP VPNv4 邻居
  • 配 VRF、RD、RT
  • 验证 VPN 路由和标签转发

排障思路：

  • 先看 IGP 通不通
  • 再看 LDP 邻居
  • 再看 MP-BGP VPNv4
  • 再看 VRF 中是否有路由
  • 最后查数据平面标签

实操案例 9：SR / SRv6 入门

目标：

  • 理解传统 MPLS 隧道与 SR 的区别
  • 先做 SR-MPLS，再看 SRv6 基础

独特技巧：

  • SR/SRv6 很容易学成“概念堆”
  • 你必须把它和“流量工程、可编程、运维简化”联系起来

13. 故障排查：高手不是会命令，而是会缩小范围

13.1 五层排障法

  1. 物理层：接口、模块、双工、速率、光功率
  2. 二层：VLAN、Trunk、MAC、STP、聚合
  3. 三层：ARP、路由、邻居、下一跳
  4. 传输层：端口、会话、ACL/NAT
  5. 应用层：DNS、HTTP、认证、NTP、API

13.2 黄金思路

  • 先确认“现象”
  • 再确认“范围”
  • 再确认“最近变更”
  • 再确认“控制平面是否正常”
  • 最后查“数据平面是否转发”

13.3 三个常见故障案例

案例 A：同 VLAN 不通

排查顺序：

  1. 网卡和接口 UP 吗
  2. VLAN 对吗
  3. 交换机 MAC 学到了吗
  4. 有环路或安全策略拦截吗

案例 B：OSPF 邻居不起

排查顺序：

  1. IP 是否可达
  2. 网络类型是否一致
  3. Hello/Dead 时间是否一致
  4. Area 是否一致
  5. 认证是否一致
  6. MTU 是否冲突

案例 C：MPLS VPN 不通

排查顺序：

  1. IGP
  2. LDP
  3. MP-BGP VPNv4
  4. VRF
  5. CE-PE 路由交互
  6. 标签转发

目标：把常见网络故障按“现象 - 可能原因 - 验证命令 - 修复动作”拆开，帮助你形成高级网络工程师的排障思维。 适用范围：华为 VRP 风格网络设备、Datacom 常见实验与项目场景。 核心原则：先确认范围，再确认分层，再确认变更，最后再确认控制面与数据面。

1. 排障总原则

1.1 不要上来就改配置

先做四件事：

  1. 记录故障现象
  2. 确认影响范围
  3. 确认最近变更
  4. 收集当前状态

建议先执行：

display interface brief
display ip interface brief
display current-configuration
display ip routing-table

1.2 分层排障法

层次核心对象典型问题
物理层接口、模块、速率、双工、光功率接口 down、丢包、CRC
二层VLAN、MAC、Trunk、STP、聚合同网段不通、广播异常
三层ARP、IP、路由、邻居跨网段不通、邻居不起
四层TCP/UDP、端口、NAT某业务不通但 ping 通
应用层DNS、HTTP、认证、API网通但业务不可用

1.3 三个关键问题

每次故障都问：

  1. 是所有业务都不通，还是某一类业务不通？
  2. 是所有用户都不通，还是某个网段/某台设备不通？
  3. 是一直不通，还是最近变更后不通？

3. 物理层故障

场景 1：接口 down/down

现象

  • display interface brief 显示接口 down
  • 无法 ping 通直连接口

可能原因

  • 网线/光纤没接好
  • 模块故障
  • 对端接口 shutdown
  • 速率/双工不匹配

验证命令

display interface brief
display interface GigabitEthernet 0/0/1

修复动作

  • 检查物理连接
  • undo shutdown
  • 检查对端接口状态
  • 统一速率与双工

专家技巧

  • 先看接口是否 up，再看协议是否 up
  • up/down 和 down/down 的含义不同

场景 2：接口 up 但丢包严重

可能原因

  • CRC 错误
  • 双工不一致
  • 光模块质量差
  • 上联拥塞

验证命令

display interface GigabitEthernet 0/0/1

重点看：

  • CRC
  • input error
  • output error
  • discard

修复动作

  • 更换线缆/模块
  • 对齐速率双工
  • 检查链路利用率

4. 二层故障

场景 3：同 VLAN 主机不通

现象

  • PC1 与 PC2 理论上同网段，但互 ping 不通

可能原因

  • 接口没进同一个 VLAN
  • 接口还是 trunk/hybrid 但接法不对
  • MAC 没学习到
  • STP 阻塞了异常端口

验证命令

display vlan
display port vlan
display mac-address
display stp brief

修复动作

  • 确认接口类型是 access
  • 确认 port default vlan
  • 确认 trunk 放行 VLAN
  • 排查是否误入隔离端口

专家技巧

  • 同 VLAN 不通先查二层，不要急着查路由
  • display mac-address 是判断二层学习是否正常的关键

场景 4：跨交换机同 VLAN 不通

可能原因

  • trunk 没放行对应 VLAN
  • trunk 两端模式不一致
  • 聚合口成员不一致

验证命令

display interface brief
display port vlan
display eth-trunk 1

修复动作

  • 在 trunk 口补 allow-pass vlan
  • 聚合口两端链路策略一致

5. 三层故障

场景 5：跨 VLAN 不通

现象

  • VLAN10 能访问本地，但访问 VLAN20 不通

可能原因

  • Vlanif 没有 IP
  • Vlanif 没有起来
  • PC 网关配置错误
  • ACL 拦截

验证命令

display ip interface brief
display current-configuration interface Vlanif 10
display current-configuration interface Vlanif 20
display acl all

修复动作

  • 给 Vlanif 配 IP
  • 确保对应 VLAN 有活跃端口，Vlanif 才会 up
  • 修正终端网关
  • 检查接口 ACL

场景 6：静态路由配置了但不通

可能原因

  • 下一跳不可达
  • 返回路由缺失
  • 路由掩码写错

验证命令

display ip routing-table
ping x.x.x.x
tracert x.x.x.x

修复动作

  • 确保下一跳直连可达
  • 补对端回程路由
  • 检查掩码

专家技巧

  • 路由“去得了但回不来”非常常见
  • 任意互通问题都要双向看

6. OSPF 故障

根据华为官方 display ospf peer、display ospf interface 等命令，OSPF 排障优先看邻居关系和接口参数。citeturn0search0

场景 7：OSPF 邻居不起

可能原因

  • IP 不通
  • Area 不一致
  • Hello/Dead 时间不一致
  • 认证不一致
  • MTU 不一致
  • Router ID 冲突

验证命令

display ospf peer
display ospf interface
display current-configuration | include ospf
display ip interface brief

修复动作

  • 确认底层三层可达
  • 对齐 Area
  • 对齐定时器
  • 对齐认证
  • 对齐 MTU 或做相应调整

排障顺序

  1. 接口 up 吗
  2. IP 通吗
  3. 邻居状态是什么
  4. 区域和认证对吗
  5. MTU 和网络类型对吗

场景 8：OSPF 邻居 Full，但没学到路由

可能原因

  • network 未宣告
  • 路由过滤
  • 汇总影响
  • 引入缺失

验证命令

display ospf routing
display ip routing-table protocol ospf
display current-configuration | include import-route

修复动作

  • 补正确网段宣告
  • 检查 area 设计与汇总
  • 检查路由策略

7. BGP 故障

场景 9：BGP 邻居建立失败

可能原因

  • 对端 AS 配错
  • 源地址不对
  • 对端不可达
  • Loopback 建邻未指定源接口
  • TCP 179 被拦截

验证命令

display bgp peer
display current-configuration | include bgp
display ip routing-table
ping x.x.x.x

修复动作

  • 修正 peer x.x.x.x as-number
  • 如果是 loopback 建邻，补 connect-interface
  • 确保有到对端 loopback 的路由

场景 10：BGP 邻居起了，但没路由

可能原因

  • 本地没宣告 network
  • 宣告前缀不在路由表
  • route-policy 过滤掉了
  • next-hop 不可达

验证命令

display bgp routing-table
display ip routing-table
display current-configuration | include route-policy

修复动作

  • 确保要宣告的前缀先存在于本地路由表
  • 放开错误策略
  • 修正 next-hop

专家技巧

  • BGP 的路由问题，很多时候不是“协议没起”，而是“策略把路由改没了”

8. MPLS / LDP / VPN 故障

场景 11：LDP 邻居不起

可能原因

  • 接口未开启 MPLS/LDP
  • IGP 不通
  • LSR-ID 问题

验证命令

display mpls ldp session
display mpls lsp
display ip routing-table

修复动作

  • 在接口上补 mpls、mpls ldp
  • 确保 IGP 打通

场景 12：MPLS VPN 不通

可能原因

  • VRF 未绑定接口
  • RD/RT 配错
  • MP-BGP VPNv4 邻居没建好
  • CE-PE 没学到客户路由
  • 标签不通

验证命令

display ip vpn-instance
display bgp vpnv4 all peer
display bgp vpnv4 all routing-table
display ip routing-table vpn-instance VPN-A
display mpls forwarding-table

修复动作

  • 检查接口绑定 VRF
  • 检查 RT import/export
  • 检查 MP-BGP 激活
  • 检查 PE-CE 路由学习

专家技巧

  • VPN 不通不要乱查
  • 固定顺序：IGP -> LDP -> MP-BGP -> VRF -> 标签

9. NAT / DHCP / Internet 出口故障

场景 13：内网能到网关，但不能上网

可能原因

  • 缺省路由没配
  • NAT 没生效
  • ACL 没匹配内网地址
  • 出口接口没做 NAT

验证命令

display ip routing-table
display nat session all
display nat outbound
display acl 2001

修复动作

  • 补默认路由
  • 检查 NAT 调用 ACL
  • 检查出口口是否调用 NAT

场景 14：终端拿不到 DHCP 地址

可能原因

  • DHCP 未开启
  • 地址池写错
  • 网关接口未做 DHCP 选择
  • 中继缺失

验证命令

display ip pool
display dhcp server ip-in-use
display current-configuration | include dhcp

修复动作

  • 开启 dhcp enable
  • 修正地址池
  • 接口补 dhcp select interface

10. 无线 / 园区常见问题

场景 15：无线已关联，但无法上网

可能原因

  • DHCP 失败
  • VLAN 映射错误
  • Portal/认证未通过
  • 出口 NAT 或 DNS 问题

验证思路

  1. 客户端有地址吗
  2. 网关能通吗
  3. DNS 能解析吗
  4. 是否通过认证

典型命令方向

  • 查看地址池使用
  • 查看 VLAN/Vlanif
  • 查看认证状态
  • 查看出口 NAT

场景 16：用户漫游后业务异常

可能原因

  • 策略未随用户迁移
  • 认证状态同步异常
  • Overlay/EVPN 控制面不稳定

专家技巧

  • 园区问题不要只看一台接入交换机
  • 要看“用户身份、位置、业务策略、控制器状态、Overlay 路径”

11. 性能与质量问题

场景 17：ping 通，但业务很慢

可能原因

  • 丢包
  • MTU 问题
  • QoS 没生效
  • 路由绕行
  • CPU/接口拥塞

验证命令

display interface GigabitEthernet 0/0/1
display cpu-usage
display ip routing-table

修复动作

  • 看接口丢弃与利用率
  • 检查大流量是否挤占链路
  • 验证路径是否绕远
  • 检查队列/QoS 策略

场景 18：高峰期偶发中断

常见思路

  • 查链路是否拥塞
  • 查 STP/OSPF/BGP 是否频繁重收敛
  • 查设备 CPU/内存是否打满
  • 查是否有环路或广播风暴

12. 变更后的故障

场景 19：改完配置后全网部分异常

高级处理流程

  1. 先冻结变更
  2. 记录故障时间点
  3. 对比变更前后配置
  4. 确认是否需要回退

建议命令

display current-configuration
display logbuffer
display saved-configuration

专家技巧

  • 网络故障很多不是技术不懂，而是变更控制差
  • 高级工程师一定要有“变更前基线、变更中验证、变更后回退预案”

14. 高级专家的故障思维

14.1 永远先问“最小闭环”

例如跨 VLAN 不通，不要一上来就查全网。 先看：

  • PC IP 对不对
  • 网关 Vlanif 对不对
  • 同交换机是否可达
  • 再扩大到 trunk、路由、策略

14.2 学会“缩圈”

排障不是一次定位，是不断缩小问题范围：

  1. 是主机问题还是网络问题
  2. 是接入问题还是骨干问题
  3. 是单点问题还是全局问题
  4. 是控制面问题还是数据面问题

14.3 高级技巧

  • 任何故障都先看“最近变更”
  • 任何协议都要有“邻居、路由、转发”三视角
  • 任何业务都要有“去程、回程”双向思维
  • 任何现象都要区分“完全不通”和“部分不通”

16. 结语

真正的高级网络专家，不是“知道很多命令”，而是：

  • 看到现象就知道从哪一层切入
  • 不被表象带偏
  • 能快速缩小范围
  • 能把排障过程讲清楚
  • 能把临时修复转化成长期优化

学网络到后面，拼的是：

  • 方法
  • 稳定性
  • 结构化思维
  • 复盘能力

如果你把这份手册配合实验一天一天过，你会明显从“会配设备”进入“会排网络问题”的阶段。

目标：把前面分散的知识点组合成综合实验，训练你接近 HCIE 的设计、部署、验证、排障能力。

案例 1：双交换机园区基础网

场景

  • 两台交换机 SW1、SW2
  • VLAN10 办公网，VLAN20 访客网
  • 三层交换网关放在 SW1

任务

  • 创建 VLAN
  • 配置 trunk
  • 配置 Vlanif 网关
  • VLAN10 与 VLAN20 可控互通

验证

  • 同 VLAN 可通
  • 跨 VLAN 可通
  • trunk 正常转发

进阶扩展

  • 增加 ACL，限制访客网访问办公网

案例 2：三路由器 OSPF 多区域

拓扑

拓扑示意自动从文档中的结构文本生成
AR1 --- AR2 --- AR3
  • AR1/AR2 在 Area 0
  • AR3 在 Area 1

任务

  • 建 OSPF 邻居
  • 发布 Loopback
  • 做 Area 划分
  • 发布缺省路由

验证

  • 邻居 Full
  • 所有 Loopback 路由可见
  • 缺省路由正常下发

故障注入

  • 故意把 AR3 配成错误 Area
  • 排查并修复

案例 3：企业双出口 BGP

场景

  • 企业 AS65001
  • ISP-A AS65002
  • ISP-B AS65003

任务

  • 与两个 ISP 建 EBGP
  • 主出口走 A，备份走 B
  • 某些路由出向优先 A
  • 入向用 AS-Path prepend 影响

验证

  • BGP 邻居 Established
  • 出口选路符合预期
  • 路由断开时切到备链路

提升点

  • 增加 route-policy 和 community 设计

案例 4：MPLS L3VPN

拓扑

CE1 -- PE1 == P == PE2 -- CE2

任务

  • 核心跑 IGP
  • 开启 MPLS/LDP
  • PE 间建立 MP-BGP VPNv4
  • 建立 VPN-A
  • CE1 与 CE2 私网互通

验证

  • LDP 邻居正常
  • MPLS 转发表正常
  • VPN 路由正常
  • 业务互通

故障注入

  • 改错 RT
  • 观察 VPN 路由不导入

案例 5：园区三层架构设计

场景

  • 接入层两台交换机
  • 汇聚层双机冗余
  • 核心层双上联
  • 业务网、访客网、管理网

任务

  • 地址规划
  • VLAN 规划
  • 网关位置设计
  • STP 根桥设计
  • Eth-Trunk 上联

验证

  • 用户接入正常
  • 上联链路故障时业务恢复
  • 访客与办公隔离

专家思考

  • 广播域是否合理
  • 是否有单点
  • 地址是否预留

案例 6：SD-WAN 业务选路方案分析题

  • 总部 + 3 分支

输出

  • 拓扑图

案例 7：自动化巡检

  • 管理 20 台设备

  • 版本
  • 接口状态
  • OSPF 邻居

案例 8：HCIE 风格综合实验

目标能力

  • 二层
  • 三层
  • 路由协议
  • BGP
  • MPLS VPN
  • 基础安全
  • 自动化验证

实验流程

  1. 先画拓扑
  2. 做地址规划
  3. 配基础接口
  4. 配二层
  5. 配三层
  6. 配 OSPF/BGP/MPLS
  7. 配策略
  8. 验证业务
  9. 故障注入
  10. 复盘

验收标准

  • 限时完成
  • 业务全部通
  • 故障能快速定位
  • 配置可解释

综合实验训练方法

方法 1：每个实验都做三个版本

  • 标准版：按题做通
  • 变形版：改需求再做一遍
  • 故障版：故意弄错再排障

方法 2：实验后必须输出

  • 拓扑图
  • 地址表
  • 配置文件
  • 验证截图或命令输出
  • 故障复盘

方法 3：限时训练

  • 基础实验：30 分钟
  • 中级实验：60-90 分钟
  • 综合实验：120-180 分钟

最后建议

HCIE 训练核心不是“做完实验”，而是：

  • 你能不能解释每一条配置为什么存在
  • 你能不能在故障时不慌
  • 你能不能从实验走向工程

3. 脚本 1：批量登录并执行命令

import csv
import paramiko

COMMAND = "display ip interface brief\n"

with open("devices.csv", newline="", encoding="utf-8") as f:
    reader = csv.DictReader(f)
    for row in reader:
        host = row["host"]
        username = row["username"]
        password = row["password"]
        print(f"== {host} ==")
        try:
            ssh = paramiko.SSHClient()
            ssh.set_missing_host_key_policy(paramiko.AutoAddPolicy())
            ssh.connect(host, username=username, password=password, timeout=5)
            shell = ssh.invoke_shell()
            shell.send(COMMAND)
            output = shell.recv(65535).decode("utf-8", errors="ignore")
            print(output)
            ssh.close()
        except Exception as e:
            print(f"{host} failed: {e}")

作用

  • 批量执行 display 命令
  • 获取基本接口状态

5. 脚本 3：巡检 OSPF 邻居

import csv
import paramiko

def get_output(host, username, password, command):
    ssh = paramiko.SSHClient()
    ssh.set_missing_host_key_policy(paramiko.AutoAddPolicy())
    ssh.connect(host, username=username, password=password, timeout=5)
    shell = ssh.invoke_shell()
    shell.send(command + "\n")
    output = shell.recv(65535).decode("utf-8", errors="ignore")
    ssh.close()
    return output

with open("devices.csv", newline="", encoding="utf-8") as f:
    reader = csv.DictReader(f)
    for row in reader:
        output = get_output(row["host"], row["username"], row["password"], "display ospf peer")
        if "Full" in output:

2. 高频面试题

2.1 基础原理题

问：VLAN 的本质是什么？

答题框架：

  • VLAN 的本质是逻辑划分广播域
  • 它不改变物理拓扑，但改变二层广播边界
  • 典型作用是隔离业务、控制广播、方便管理
  • 真正工程价值在于：把用户/业务和物理端口解耦

问：为什么要用 STP？

答题框架：

  • 二层冗余会带来环路
  • 环路会导致广播风暴、MAC 地址表震荡、重复报文
  • STP 通过阻塞部分链路，构建无环拓扑
  • 工程上要结合 Eth-Trunk、MSTP、网关位置一起看

问：静态路由和 OSPF 的差别？

答题框架：

  • 静态路由简单、可控、开销低，但扩展性差
  • OSPF 适合中大型内部网络，支持动态收敛和层级设计
  • 小网络可优先静态，大中型园区和骨干通常使用 OSPF

3. BGP / MPLS / EVPN 高频题

问：为什么企业双出口一般会用 BGP？

答题框架：

  • 多出口场景需要更灵活的路径控制
  • BGP 可以通过 Local Preference、AS-Path、MED、Community 控制选路
  • 它适合大规模和策略化路由控制
  • 真正价值不是“能通”，而是“可运营、可调优、可扩展”

问：MPLS L3VPN 为什么要 RD 和 RT？

答题框架：

  • RD 用来保证 VPNv4 路由全局唯一
  • RT 用来控制路由导入导出策略
  • RD 解决“唯一性”，RT 解决“策略性”

问：为什么现在很多园区从传统 VLAN 演进到 VXLAN/EVPN？

答题框架：

  • 传统园区广播域和策略容易受物理拓扑限制
  • VXLAN/EVPN 支持 Overlay 封装，提升扩展性
  • 更适合多园区、虚拟化、统一策略和自动化运维
  • 重点不只是“新技术”，而是解决原架构痛点
