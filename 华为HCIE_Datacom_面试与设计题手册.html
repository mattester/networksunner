1. 高级网络岗位最看重什么

面试官通常不是只看你会不会命令，而是看你是否具备：

  • 原理理解能力
  • 架构设计能力
  • 故障排查能力
  • 项目交付能力
  • 自动化与运维思维
  • 沟通与表达能力

回答任何问题，尽量按这个顺序：

  4. 再说风险
  5. 再说验证与运维

2. 高频面试题

2.1 基础原理题

问：VLAN 的本质是什么？

答题框架：

  • VLAN 的本质是逻辑划分广播域
  • 它不改变物理拓扑，但改变二层广播边界
  • 典型作用是隔离业务、控制广播、方便管理
  • 真正工程价值在于：把用户/业务和物理端口解耦

问：为什么要用 STP？

答题框架：

  • 二层冗余会带来环路
  • 环路会导致广播风暴、MAC 地址表震荡、重复报文
  • STP 通过阻塞部分链路，构建无环拓扑
  • 工程上要结合 Eth-Trunk、MSTP、网关位置一起看

问：静态路由和 OSPF 的差别？

答题框架：

  • 静态路由简单、可控、开销低，但扩展性差
  • OSPF 适合中大型内部网络，支持动态收敛和层级设计
  • 小网络可优先静态，大中型园区和骨干通常使用 OSPF

3. BGP / MPLS / EVPN 高频题

问：为什么企业双出口一般会用 BGP？

答题框架：

  • 多出口场景需要更灵活的路径控制
  • BGP 可以通过 Local Preference、AS-Path、MED、Community 控制选路
  • 它适合大规模和策略化路由控制
  • 真正价值不是“能通”，而是“可运营、可调优、可扩展”

问：MPLS L3VPN 为什么要 RD 和 RT？

答题框架：

  • RD 用来保证 VPNv4 路由全局唯一
  • RT 用来控制路由导入导出策略
  • RD 解决“唯一性”，RT 解决“策略性”

问：为什么现在很多园区从传统 VLAN 演进到 VXLAN/EVPN？

答题框架：

  • 传统园区广播域和策略容易受物理拓扑限制
  • VXLAN/EVPN 支持 Overlay 封装，提升扩展性
  • 更适合多园区、虚拟化、统一策略和自动化运维
  • 重点不只是“新技术”，而是解决原架构痛点

示例：总部 + 分支园区网络设计

题目

某企业有 1 个总部、5 个分支、3000 用户，需要设计园区与广域一体化网络。

标准答法

业务背景：

  • 总部有办公、研发、视频会议、访客接入
  • 分支需要访问总部业务和 Internet

现网问题：

  • 传统二层扩展复杂
  • 多分支管理成本高
  • 认证、策略不统一

约束条件：

  • 预算有限
  • 需要分阶段割接
  • 必须高可用

方案建议：

  • 园区采用三层架构或 VXLAN/EVPN 架构
  • 广域采用 SD-WAN 或 MPLS+Internet 混合
  • 总部统一认证与策略下发
  • 核心节点双机冗余

风险点：

  • 地址规划冲突
  • 策略变更影响用户
  • 广域链路切换异常

验证方法：

  • 连通性验证
  • 路由验证
  • 认证验证
  • 故障切换验证

运维建议：

  • 建立配置基线
  • 自动化巡检
  • 日志与性能可视化

5. 设计题里的取舍思维

5.1 地址规划怎么答

优秀答案不是只给网段，而是要说明：

  • 按组织还是按楼层规划
  • 是否预留扩容空间
  • 是否为管理、无线、访客独立规划

5.2 网关位置怎么答

常见说法：

  • 小规模接入可放汇聚
  • 大型园区或 Overlay 更倾向网关下沉或分布式网关
  • 关键是看流量模型、收敛需求和管理复杂度

5.3 为什么不用单一大二层

答法：

  • 广播域过大
  • 收敛和故障域不可控
  • 运维复杂
  • 不利于规模扩展

6. 面试时怎么讲项目

推荐结构

  1. 项目背景
  2. 你的职责
  3. 网络规模
  4. 技术难点
  5. 你的设计思路
  6. 你怎么验证
  7. 出过什么问题，你怎么解决
  8. 项目复盘

示例表达

不要说：

  • “我配了 OSPF、BGP、ACL”

要说：

  • “项目中总部与分支双出口场景需要主备和业务分流，我采用 OSPF 承载内部路由、BGP 处理出口选路，并通过策略控制视频流量优先，最终把链路切换时延压缩到业务可接受范围。”

7. 高级表达技巧

技巧 1：把技术词翻译成业务词

  • OSPF 收敛快 -> 业务恢复快
  • VXLAN/EVPN -> 架构更易扩展
  • 自动化巡检 -> 降低人工风险

技巧 2：回答时要有取舍

面试官更喜欢听到：

  • 为什么选 A，不选 B
  • A 的优势是什么，代价是什么

技巧 3：把失败讲出来

高级工程师不是没出过问题，而是：

  • 发现得快
  • 定位得准
  • 复盘得清
